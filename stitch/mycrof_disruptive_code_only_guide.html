# MYCROF: Digital Art & Code-Only Development Guide (The Disruptive Path) This document replaces previous WordPress-focused guides. We are moving to a **Code-First, High-Performance Art** stack to achieve spectacular animations and a truly unique digital experience. ## 1. Core Tech Stack (The "No-Build" Freedom) - **Frontend Framework:** Next.js 14 (App Router) or Vite + Vanilla TS. - **Rendering Engine:** WebGL via **Three.js** and **React Three Fiber**. - **Animation Core:** **GSAP** (GreenSock) with ScrollTrigger, Flip, and CustomEase. - **Styling:** Tailwind CSS (for layout) + Custom GLSL Shaders (for backgrounds and text effects). - **Smooth Scroll:** Lenis or GSAP ScrollSmoother. ## 2. Visual & Interaction Architecture ### A. The "Digital Void" Hero - **Background:** A 3D particle system (MYCROF logo shape) that reacts to cursor proximity using a custom fragment shader. - **Typography:** Kinetic text that "breaks" or glitches into pixels on scroll. - **Interaction:** No standard buttons. CTAs are magnetic areas with a "glitch-glow" effect. ### B. Liquid Transitions - Use a displacement map to create "liquid" transitions between sections. As the user scrolls, the page should look like it's being distorted by a wave. - **Shader Instruction:** Implement a `displacementMap` using Perlin Noise to warp the entire viewport during scroll-jumps. ### C. Generative Content Cards - Instead of static grids, use a "Generative Layout" where elements drift slightly based on device orientation (Gyroscope/Mouse Parallax). - Cards should have a "Glass-Hologram" effect: `backdrop-filter: blur(40px)` combined with a shifting chromatic aberration on the borders. ## 3. Implementation Checklist for Developers 1. **Global Shader Overlay:** Create a `post-processing` layer for the whole site that adds a subtle film grain and chromatic aberration at the screen edges. 2. **Custom Text Plugin:** Use GSAP's `SplitText` to animate every single letter with different delays and 3D rotations. 3. **Data Visualization (The Network):** Represent "Talent" as a live-rendering node graph (Canvas API) that connects and disconnects dynamically. 4. **Zero-Latency Navigation:** Implement a Single Page Application (SPA) structure with "Page Transitions" using GSAP's `Flip` to morph elements from one view to the next. ## 4. Performance Optimization (The "Art" of Speed) - Use **OffscreenCanvas** for heavy WebGL computations. - Optimize 3D assets using **gltf-pipeline** and Dracho compression. - Implement strict "Lazy-Hydration" for non-critical visual components. --- *Forget templates. This is a digital experience built from the ground up.*
